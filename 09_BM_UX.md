# 09. 수익 모델 (BM) / UX·UI 방향

> 설계 원칙
> - "복잡한 시스템, 단순한 UI" — 내부는 20개 이상 변수, 플레이어에게는 5개 스탯 + 등급
> - "Pay to Win 금지" — 과금은 체험의 확장이지, 실력의 대체가 아니다
> - "선택의 무게를 훼손하지 않는다" — 선택 리셋 직접 판매 금지

---

## 1. 수익 모델 (BM)

### 1-1. 핵심 원칙

```
[Pay to Experience]
과금 = 더 많은 체험, 더 다양한 이야기
과금 ≠ 더 높은 스탯, 더 쉬운 게임

이 게임의 핵심은 "선택의 무게"와 "포기의 고통".
돈으로 고통을 사라지게 하면 게임의 의미가 사라진다.
```

### 1-2. 수익 항목

| 항목 | 유형 | 가격대 | 설명 |
|------|------|--------|------|
| **코스튬** | 소액 반복 | 1,000~3,000원 | 캐릭터 외형, 온라인 프로필 반영, 스탯 영향 없음 |
| **시나리오 DLC** | 중액 1회 | 5,000~10,000원 | 학교 종류 추가 (특목고, 예술고 등) — 리플레이 핵심 |
| **평행 세계** | 중액 1회 | 3,000~5,000원 | 특정 분기점에서 "다른 선택"을 체험. 본 세이브 영향 없음 |
| **앨범 확장** | 소액 1회 | 1,000~2,000원 | 히든 앨범 카드 추가, 특수 회고 장면 해금 |
| **본편** | **무료 (F2P)** | 0원 | 기본 플레이 전체 포함. 웹 접속 즉시 플레이 |

### 1-3. 절대 하지 않는 것

| 금지 항목 | 이유 |
|-----------|------|
| 선택 리셋 판매 | 게임 핵심("선택의 무게") 훼손 |
| 스탯 부스트 | "포기의 고통" 무효화 |
| 에너지/턴 제한 | 이 게임은 기다림이 아닌 몰입이 핵심 |
| 가챠/랜덤 박스 | 학교물 분위기와 맞지 않음 |
| 광고 보고 스탯 획득 | Pay to Win의 변형 |
| 본편 유료화 | 웹 게임은 무료 접근이 생명. 진입장벽 = 유저 이탈 |

### 1-4. 코스튬 상세

```
[코스튬 시스템]

적용 범위:
- 캐릭터 일러스트 (메인 화면, 이벤트 컷신)
- 온라인 프로필 (피드/스토리에 반영)
- NPC 반응: 첫 착용 시 "오 새 옷?" 코멘트 (친밀도 +1, 1회)

코스튬 종류:
- 교복 커스텀 (악세서리, 가디건, 신발 등)
- 사복 (주말/방학 활동 시 표시)
- 특수 코스튬 (시즌 한정, 이벤트 보상)

핵심: 순수 비주얼. 스탯/성장/이벤트 분기에 영향 없음.
NPC 반응도 "코멘트 1줄" 수준으로 미미하게.
```

### 1-5. 시나리오 DLC 상세

```
[DLC = 새로운 학교, 새로운 인생]

각 학교 DLC는 사실상 "다른 게임 모드":
- 슬롯 구조가 다름
- NPC 풀이 다름 (일부 겹침)
- 등수 분포가 다름
- 전용 이벤트/딜레마
- 전용 엔딩 분기

계획:
- DLC 1: 특목고 (과학고/외고) — 최우선
- DLC 2: 예술고/체육고 — 재능 특화 빌드 대상
- DLC 3: 특성화고 (공업/상업) — 자립 루트 강화

리플레이 구조: 본편(일반고) 1회 → DLC(다른 학교) → "같은 인생, 다른 학교"
```

### 1-6. 평행 세계 모드

```
[평행 세계 = "그때 다른 선택을 했다면?"]

기능:
- 플레이 중 특정 분기점을 "북마크"
- 엔딩 후, 북마크 지점으로 돌아가 다른 선택을 체험
- 본 세이브에 영향 없음 (평행 세계는 별도 기록)
- 평행 세계 결과도 앨범에 기록 ("다른 나의 인생" 탭)

제한:
- 무료: 엔딩 후 1개 분기점 체험 가능
- 유료: 추가 분기점 해금 (3,000~5,000원에 전체 해금)

설계 의도:
- "선택 리셋"을 직접 팔지 않으면서도 "다른 선택" 욕구를 충족
- 본 플레이의 선택은 영원히 남음 — 평행 세계는 "만약에"일 뿐
- 후회 카드에서 "그때 다르게 했으면..." → 자연스러운 과금 동선
```

## 2. UI 설계 방향

### 2-1. 핵심 원칙

```
[복잡한 시스템, 단순한 UI]

내부: 20개 이상 변수 (5스탯 + 하위스킬 + 피로 + 멘탈 상태 + 친밀도 + 가시성 + ...)
플레이어: 5개 스탯 등급 + 용돈 + 관계 상태 + 현재 주차

"숫자를 보여주지 말고, 학교생활을 보여줘라."
```

### 2-3. UI 상태 표현 체계

```
[스탯은 숫자가 아니라 "느낌"으로]

수치 직접 노출 대신:
- 등급 (E/D/C/B/A) + 색상 (회색/빨강/노랑/초록/금색)
- 마일스톤 진행 바 (다음 마일스톤까지 얼마나 남았는가)
- 컨디션 아이콘 (😊 좋음 / 😐 보통 / 😰 피곤 / 😱 번아웃)

피로는 단계로:
- 좋음 (0~19): 초록 아이콘
- 경미 (20~34): 노랑 아이콘
- 주의 (35~49): 주황 아이콘 + 경고 텍스트
- 위험 (50~69): 빨강 아이콘 + "쉬어야 합니다"
- 극한 (70+): 빨강 깜빡임 + "번아웃 위험!"

멘탈은 표정으로:
- 80+: 밝은 표정
- 50~79: 보통 표정
- 30~49: 어두운 표정
- 30-: 무표정/눈물
- 번아웃: 화면 채도 저하

관계는 거리와 색으로:
- 친밀도 80+: 가까이, 초록 선, 하트 아이콘
- 50~79: 보통 거리, 노란 선
- 30~49: 먼 거리, 회색 선
- 30-: 매우 먼 거리, 점선 (거의 안 보임)
```

### 2-4. 정보 공개 원칙

```
[점진적 정보 공개 — Progressive Disclosure]

Y1 (초등 6학년):
- 보이는 것: 학업, 인기, 체력 (3개만)
- 안 보이는 것: 재능, 멘탈 (아직 의미 없음)
- 피로: "컨디션" 정도로만 표시

Y2 (중학교 진학):
- 재능 스탯 공개 ("좋아하는 게 뭔지 알겠다")
- 멘탈 스탯 공개 ("요즘 마음이 복잡하다")
- 온라인 시스템 해금
- 피로 단계 표시 시작

Y5 (고등학교 진학):
- 내부 수치 일부 공개 (등수, 생활기록부 점수)
- 진학 시뮬레이터 해금 ("이 성적이면 어디 갈 수 있을까?")
- 수능 예상 등급 표시

원칙: 시스템은 처음부터 작동하지만, 보여주는 건 천천히.
"아, 이런 것도 있었구나?" 하는 발견의 재미.
```

### 2-5. 조작 체계


## 5. 아트 파이프라인 — AI 생성 기반 일관성 확보

> 원칙
> - 예산 제한 하에서 **상업 품질에 근접하는** 비주얼을 확보한다
> - AI 생성 + 수동 보정으로 캐릭터 일관성 80~90% 수준을 목표로 한다
> - EA(Early Access) 단계에서 시작하고, 수익 발생 후 프로 일러스트로 업그레이드 가능한 구조로 설계한다
> - Steam AI 콘텐츠 공개 의무(2024~) 준수

### 5-1. 필요한 그래픽 리소스 목록

| 카테고리 | 내용 | 수량 (추정) | AI 적합도 |
|----------|------|------------|-----------|
| **캐릭터 일러스트** | NPC 10명 + 주인공(남/여) × 표정 5~8종 | 60~100장 | 중 (일관성 작업 필요) |
| **배경** | 교실, 복도, 학원, 집, 공원, 카페, 도서관 등 | 15~20장 | 높음 (일관성 유지 쉬움) |
| **컷신 CG** | 핵심 이벤트 장면 | 20~30장 | 중~높음 |
| **UI 아이콘** | 스탯, 활동 카드, 상태 표시, 감정 아이콘 | 50~80개 | 낮음 (직접 제작 권장) |
| **코스튬** | 교복 변형, 사복 (BM 아이템) | 확장형 | 중 |
| **배경 오버레이** | 계절, 시간대, 시험 기간, 번아웃 등 분위기 | 10~15종 | 높음 |

### 5-2. 캐릭터 일관성 확보 파이프라인

```
[STEP 1: 캐릭터 디자인 시트 — 가장 중요한 단계]

각 NPC마다 "기준 프롬프트"를 확정한다.
이 프롬프트가 전체 일관성의 기반이 된다.

예시 — 지훈 (#1 소꿉친구):

  기본 프롬프트 (고정):
  "korean teenage boy, 16 years old, short messy black hair,
   warm brown eyes, slightly tan skin, athletic build,
   korean school uniform navy blazer white shirt,
   anime/manhwa style, soft cel-shading, clean lineart,
   upper body portrait, simple pastel background,
   consistent character design sheet"

  스타일 앵커 (전 캐릭터 공통):
  "manhwa style, soft cel-shading, clean lineart,
   pastel color palette, school life theme,
   high quality, detailed face"

핵심: "스타일 앵커"를 모든 캐릭터에 동일하게 적용하여
      다른 인물이지만 "같은 세계관" 느낌을 유지한다.
```

```
[STEP 2: 기준 이미지 확정 — 한 캐릭터당 1장]

1. 기본 프롬프트로 10~20장 생성
2. 가장 마음에 드는 1장을 "기준 이미지"로 선정
3. 이 이미지의 seed 값, 파라미터를 기록/저장
4. 모든 후속 이미지는 이 기준에서 파생

기록 양식:
├─ 캐릭터명: 지훈
├─ 기준 이미지: jihun_base_v3.png
├─ 모델: [사용한 AI 모델명]
├─ 프롬프트: [전체 프롬프트]
├─ 네거티브: [네거티브 프롬프트]
├─ Seed: [시드값]
├─ CFG Scale: [값]
├─ Steps: [값]
└─ 비고: "눈이 약간 크게 나옴 — 후보정으로 조정"
```

```
[STEP 3: 표정 변형 — img2img 또는 참조 이미지 활용]

기준 이미지를 입력으로 넣고, 표정/감정만 변경한다.

방법 A — img2img (Stable Diffusion 계열):
- 기준 이미지를 img2img에 넣고 denoising 0.3~0.5로 설정
- 프롬프트에 표정만 변경: "bright smile" → "sad expression" → "angry face"
- 낮은 denoising = 원본 유지도 높음, 표정 변화는 작음
- 높은 denoising = 표정 변화 크지만, 외형도 변할 수 있음
- 적정값: 0.35~0.45 (표정은 바뀌되 인물은 유지)

방법 B — 참조 이미지 (Midjourney, DALL-E 등):
- "--cref [기준이미지 URL]" (Midjourney Character Reference)
- 기준 이미지를 참조로 주고 표정/상황만 변경
- Midjourney의 --cref는 일관성이 비교적 높음

방법 C — ControlNet (Stable Diffusion):
- OpenPose로 포즈 고정 + 얼굴 참조로 표정만 변경
- 기술적 난이도 높지만 일관성 최고

NPC당 필요 표정:
├─ 기본 (무표정/자연스러움)
├─ 웃음 (밝은)
├─ 슬픔 (우울/눈물)
├─ 화남 (짜증/분노)
├─ 놀람 (당황)
├─ 부끄러움 (연애 이벤트용)
├─ 진지함 (심각한 대화용)
└─ 지침 (피로/번아웃용) — 선택적
```

```
[STEP 4: 후보정 — 일관성의 마지막 보루]

AI가 90%를 만들고, 사람이 10%를 다듬는다.

필수 보정 항목:
├─ 눈동자 색상 통일 (같은 캐릭터인데 색이 미묘하게 다른 경우)
├─ 머리 스타일/색상 통일 (앞머리 방향, 길이 등)
├─ 교복 디테일 통일 (넥타이 색, 블레이저 버튼 수 등)
├─ 피부톤 통일 (조명에 따라 달라지는 경우)
├─ 배경 제거/통일 (투명 배경 또는 공통 배경으로)
└─ 크기/위치 정규화 (게임 내 동일한 위치에 표시되도록)

도구: Photoshop, GIMP(무료), Photopea(무료 웹), 또는 Clip Studio Paint
작업 시간: 1장당 5~15분 (간단한 색상/디테일 보정)
```

```
[STEP 5: 학년별 성장 — 7년간의 변화]

이 게임은 초6~고3까지 7년을 다루므로, 캐릭터가 성장해야 한다.

학년 그룹 (3단계로 나누어 리소스 절약):
├─ 초등~중1 (Y1~Y2): 어린 버전 (둥근 얼굴, 작은 체형)
├─ 중2~고1 (Y3~Y5): 성장 버전 (얼굴 갸름해짐, 키 성장)
└─ 고2~고3 (Y6~Y7): 성인에 가까운 버전

각 단계별로 기준 이미지를 새로 만들되,
프롬프트의 "스타일 앵커"는 동일하게 유지하여 같은 인물로 인식되게 한다.

프롬프트 변형 예시 (지훈):
- Y1: "13 year old korean boy, round face, short, childish features..."
- Y4: "16 year old korean boy, sharper jawline, taller, mature features..."
- Y7: "18 year old korean young man, defined jawline, tall, near-adult features..."
```

### 5-3. 배경 생성 파이프라인

```
[배경은 캐릭터보다 훨씬 쉬움]

1. 장소별 기본 프롬프트 확정
   "korean school classroom, empty desks, afternoon sunlight,
    anime background style, soft pastel colors, no characters"

2. 시간대/계절 변형
   기본 배경 1장 → img2img로 아침/점심/저녁/밤 4종 생성
   봄(벚꽃)/여름(초록)/가을(단풍)/겨울(눈) 4종 생성
   = 1장소 × 4시간 × 4계절 = 최대 16종 (필요한 것만 선별)

3. 분위기 오버레이
   시험 기간: 약간 어두운 색감 + 긴장감
   번아웃: 채도 저하 + 흐릿함
   축제/이벤트: 밝고 화려한 색감
   → CSS 필터로 처리 가능 (배경 이미지 교체 불필요)
```

### 5-4. 컷신 CG 생성

```
[핵심 장면만 CG로 — 나머지는 캐릭터+배경+텍스트 연출]

CG가 필요한 장면 (우선순위):
├─ 게임 시작: 부모 선택 장면
├─ 졸업식 (초등/중학/고등 각 1장)
├─ 수능 당일 아침
├─ 수능 전날 통화 장면
├─ 엔딩 CG (메인 엔딩별 1장씩)
└─ 핵심 딜레마 장면 (선택적)

- 캐릭터 기준 프롬프트 + 상황 프롬프트 조합

- 또는 넓은 구도의 프롬프트로 생성 후 후보정

EA 단계: 컷신 10장 내외로 시작
정식 출시: 20~30장으로 확장
```

### 5-7. Steam AI 콘텐츠 공개 의무

```

- AI 생성 콘텐츠를 사용하는 게임은 스토어 페이지에 공개 의무
```
