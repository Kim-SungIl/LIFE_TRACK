# M5 세로 슬라이스 체감 체크리스트

자동 리포트(`scripts/report-m5-playthrough.ts`)로 못 잡는 영역 — UI 연출·정서·체감. 실 브라우저에서 한 번은 끝까지 플레이하며 확인.

---

## 1. 학년말 일기장 (M3.5)

- [ ] Y1 끝에 "YEAR-END" 화면이 뜨는가?
- [ ] "초등학교 6학년" 같은 학년 이름이 정확한가?
- [ ] 해당 학년의 memorySlot recallText가 표시되는가? (비어 있으면 "특별히 기억에 남는 일은 없었다")
- [ ] milestoneScene summaryText가 "돌아보면" 섹션에 나오는가?
- [ ] "다음 학년으로" 버튼으로 진행하면 다음 학년 W1로 넘어가는가?
- [ ] Y7 끝에는 year-end 스킵하고 바로 엔딩으로 가는가?
- [ ] **정서 평가**: "아, 이 학년은 이런 시간이었구나" 싶은 감각이 드는가? 밋밋하면 memorySlot이 sparse하거나 밍숭맹숭한 것.

## 2. 엔딩 회상 (M3)

- [ ] "돌아보면" 섹션에 memorialHighlights 3~5개 뜨는가?
- [ ] "7년의 마디" 섹션에 yearClosings(학년별 summaryText) 뜨는가?
- [ ] 회상 문장이 학년·사건을 특정하는가? 일반 fallback 문장만 나오면 소재 부족 신호
- [ ] **정서 평가**: 회상을 읽고 "내 캐릭터가 이렇게 살았구나" 싶은가? 기계적으로 느껴지면 recallText 품질 이슈

## 3. 주요 이벤트 발동

- [ ] Y2 W28 수학여행 (중1 경주) — 참가/불참 선택
- [ ] Y5 W28 수학여행 (고1 제주) — 참가/불참 선택
- [ ] Y1 W45 초등 졸업 앨범 — 신청/스킵
- [ ] Y7 W45 고등 졸업 준비 — 정장/평상복
- [ ] Y5 W2 동아리/학원 선택 — 학원(버프)/동아리(버프)/자율
- [ ] 주요 NPC 생일 이벤트에 "정성 선물 (-5만원)" 옵션 뜨는가?

## 4. 위기 이벤트

- [ ] Y3 근처에서 middle-burnout (중2 번아웃) 또는 minjae-jealousy (민재 질투) 같은 소프트/하드 크라이시스가 뜨는가?
- [ ] 크라이시스 이벤트 선택 후 memorySlot이 실제로 생성됐는지 (다음 학년말 일기장에 반영)

## 5. 상점·돈 싱크

- [ ] 7년 플레이 후 잔고가 500만원 이상 남지 않는가? (남으면 돈 싱크 실패 신호)
- [ ] 학원/동아리 버프 선택 시 루틴 효율이 체감되는가? (주간 스탯 상승폭 증가)

## 6. 저장·로드 호환

- [ ] 중간 저장 → 탭 닫기 → 재오픈 시 같은 지점에서 재개
- [ ] M1 이전 구세이브 있으면 로드 후 크래시 없는지 (개발자 도구로 5개 신규 필드 제거 후 로드 테스트)

## 7. 팔레트·UI

- [ ] 타이틀 화면 로고 그라데이션이 테라코타 → 골드인가?
- [ ] 선택지 호버 색이 핫핑크가 아닌 따뜻한 터라코타인가?
- [ ] EventScene 배경 그라데이션이 일관되게 warm-tone인가?
- [ ] milestone 박스의 골드 강조가 antique 톤인가? (너무 쨍한 노랑이면 CSS 변수 이슈)

## 판정 기준 (플레이 직후 바로 체크)

| 영역 | 양호 | 개선 필요 |
|---|---|---|
| memorySlot 생성 | 7년 플레이 후 6개 이상 | 3개 이하 = 소스 이벤트 확대 필요 |
| 엔딩 회상 개인화 | 회상 중 2개 이상이 특정 사건 | 전부 fallback 문장 = recallText 소재 부족 |
| milestone 개인화 | Y3 패턴 중 하나 매칭 | 전부 fallback = 크라이시스 활성화 실패 |
| 학년말 일기장 정서 | "아 이 학년 이랬지" 감각 | 밋밋하면 이벤트 참여율 저조 |
| 돈 싱크 | 잔고 300만원 이하 | 1000만원 초과 = M7 기억 매개 상점 필요 |

## 자동 리포트와 교차 확인

`npx tsx scripts/report-m5-playthrough.ts` 출력 중 **[경고 포인트]** 섹션을 먼저 본다. 경고가 없어도 체감이 나쁠 수 있으므로 이 체크리스트 병행.
