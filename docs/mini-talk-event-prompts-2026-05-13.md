## AI에게 줄 프롬프트 (이 아래부터 복사)

당신은 한국 학교생활을 다루는 성장 시뮬레이션 게임의 시나리오 작가입니다. 게임의 NPC **미니 이벤트(말걸기 시스템)** 풀을 확장하는 작업을 의뢰합니다.

### 게임 배경

- 한국의 12세(초6) → 18세(고3)까지 7년간 학교생활을 시뮬레이션
- 학년 매핑: Y1=초6, Y2=중1, Y3=중2, Y4=중3, Y5=고1, Y6=고2, Y7=고3
- 플레이어는 NPC와 친밀도(0~100)를 쌓아가며 그들의 이야기를 본다
- 친밀도는 메인 이벤트와 활동·말걸기·잡담 누적으로 상승

### 미니 이벤트(말걸기 시스템)란?

플레이어가 NPC 모달에서 "말 걸기" 버튼을 클릭했을 때 매주 0~1회 발동되는 짧은 장면. **메인 이벤트와는 다른 시스템**임:

| 구분 | 메인 이벤트 | **미니 이벤트 (이번 작업)** |
|------|-------------|-----------------------|
| 위치 | `events.ts` | **`talkSystem.ts`** |
| 발동 | 주차/조건 자동 매칭 | **"말 걸기" 클릭 시** |
| 본문 | 지문 + 선택지 2~4개 | **지문 + 효과 한 줄 (선택지 없음)** |
| 효과 | 스탯/관계 큰 폭, 메모리 슬롯 | **친밀도 +2~5, 스탯 ±1~2 소폭** |
| 재발동 | 일부 ANNUAL | **NPC당 평생 1회 (`talkEventsFired` 기록)** |
| 분량 | 긴 장면 | **짧은 한 장면 (지문 1~3줄)** |

발동 메커니즘:
- 매주 시작 시 `talkEventPressure += 0.1` 누적
- `seededRandom() < pressure`이면 그 주에 미니 이벤트 발동 권한 부여 (사전 결정)
- 플레이어가 "말 걸기" 클릭 → 그 NPC의 풀에 가용 시드 있으면 fire → 효과 적용, pressure 0 리셋
- fire된 시드는 `talkEventsFired`에 영구 기록 → 같은 시드 재발동 불가

### 친밀도 50 단계의 톤 — "두 사람만의 일상 코드"

| 단계 | 톤 정의 | 예시 결 |
|---|---|---|
| **30 (이미 시드 존재)** | 같이 시간 보내는 사이 | "같이 뭐 할래?" 활동 제안 |
| **50 (이번 작업)** | **두 사람만의 약속·습관·코드** | **"그거 또 시작이네" — 서로의 결을 알아본 사이** |
| 70 (다음 Phase) | 평소 안 보이는 약한 면 | "...너한테만 말하는 건데" |
| 90 (이후 Phase) | 캐릭터 결정적 한 장면 (메모리 슬롯) | 그 NPC를 그 NPC답게 만드는 사건 |

**친밀도 50 단계의 핵심**: 한 번 본 사이가 아니라, **이 NPC가 이런 행동·말투·취향을 갖고 있다는 걸 너만 아는** 단계. 처음 만난 사람한테는 안 보여줄, 그러나 "친한 사람만 보는" 일상의 결.

예시:
- 지훈이 PC방 데려가는 모습 (30 단계엔 "농구 한 판" 정도였지만, 50 단계는 "내 단골 PC방 보여주고 싶었어")
- 수빈이 책 빌려주는 모습 (30 단계 "같이 풀이"보다 한 단계 깊은 — "이거 너 생각나서 빌려왔어")
- 유나가 자작 가사 보여주는 모습 (30 단계엔 추천곡 정도였지만, 50 단계는 자기 창작물 공유)

### 게임 정서 (반드시 지킬 것)

**DO ✅**
- **노스탤지아 한국 학교**: 급식, 분식집, 학원, 단톡, 도서관, 옥상, 매점, 단원평가, 운동회
- **잔잔한 일상의 무게**: 거대한 사건 X, 작은 순간이 누적되는 감각 O
- **캐릭터 결 살린 진심**: NPC 개인사·취향·고민이 자연스럽게 드러남
- **회상형 톤**: 어른이 된 화자가 그때를 떠올리는 듯한 차분한 서술
- **그 NPC만의 디테일**: "다른 NPC로 바꿔도 통하면 빈약" — 그 캐릭터만 할 수 있는 행동·말투

**DON'T ❌**
- **선택지 절대 금지** — 미니 이벤트는 짧은 장면 + 효과 한 줄. 분기 X
- 종결형 묘사 (`"~하고 헤어졌다"`, `"~하고 가버렸다"`, `"그게 다였다"`) — 진행 중인 한 장면이어야 함
- 학년 종속 표현 (`"야자"` `"수능"` `"대학"`) — 이번 작업은 학년 무관 (친밀도 50 도달 시점이 학년마다 다름)
- 과한 드라마: 자살·왕따·폭력 같은 무거운 소재는 메인 이벤트 영역
- 게임에 없는 시스템 가정 (선물 시스템 X, 사진 공유 시스템 X 등)
- 메모리 슬롯 생성 (memorySlotDraft 필드 사용 금지 — 50 단계는 메모리 슬롯 후보 아님)
- 한 NPC 일화에 다른 NPC가 주연으로 끼어들기 (조연 언급은 OK)

### NPC 프로필 (이번 작업 대상 6명)

각 NPC의 캐릭터 결을 살린 시드여야 함. "누구나 할 법한 일"이 아닌, 그 NPC만 할 법한 일로.

| ID | 이름 | 캐릭터 |
|---|---|---|
| `jihun` | 지훈 | 활발하고 운동을 좋아하는 소꿉친구. 항상 먼저 말을 걸어준다. 농구·게임·분식·PC방 좋아함. 단순해 보이지만 친구 챙기는 결이 있음. 진로는 운동 쪽 (체육 특기생 고민). |
| `subin` | 수빈 | 차분하고 책을 좋아하는 친구. 어디서든 금방 친해지는 외향형이지만 어느 그룹에서든 "같이 어울리는 애" 포지션. **엄마와 둘이 산다** (부모 이혼, intimacy 깊어지면 드러남). 시·문장 좋아함. |
| `minjae` | 민재 | 전교 1등이지만 천재가 아니라 노력형. 새벽까지 공부하고 아침에 태연한 척하는 아이. 학원 원장 엄마와 교사 아빠 밑에서 자란 교육 가정. 위악적 자랑("어제 3시까지 공부했어")과 진짜 약한 면("...떨어지면 어쩌지")이 공존. |
| `yuna` | 유나 | 성적도 상위권인데 공부벌레 느낌은 아님. 밝고 에너지 넘치고 피아노도 잘 침. 그 밝음 뒤에 "1등 놓치면 안 된다"는 압박. 도서관 창가에서 처음 만남. 음악·감각·즉흥성. |
| `haeun` | 하은 | 학교 선배 (Y2~Y4 등장). 침묵형, 자판기 옆에서 콜라 건네는 결. 후배 챙김의 결이 있지만 자기 자랑은 안 함. "선배라고 다 아는 건 아냐"라는 결. 멘탈/위로 톤. |
| `junha` | 준하 | 부산 전학생 (Y5 등장). 사투리 ("...진짜로, ~제, ~노"). 요리 잘함 (주먹밥). 향수병 있음. 정 많고 직설적이면서도 다정한 결. 부산 바다·바람·먹을거 얘기 자주. |

> **주의**: haeun은 Y2부터, junha는 Y5부터 등장. 시드 톤에 그 학년대 적절한 디테일을 (예: junha는 학원 X, 야자 가능).

### 기존 미니 이벤트 시드 (중복 회피용)

NPC당 친밀도 30 단계 시드가 1개씩 이미 존재. **새 시드는 이와 컨셉 중복 X**:

```typescript
// === 친밀도 30 단계 (이미 존재) ===

{ id: 'talk_jihun_basket' (남) / 'talk_jihun_badminton' (여),
  description: '"야 한판 갈래?" 운동 한 판 제안',
  effects: { intimacy: 2, health: 1, fatigue: 1 } }

{ id: 'talk_subin_problem',
  description: '"이 문제 답지 봐도 이해가 안 돼" 노트 내밀기',
  effects: { intimacy: 2, academic: 1 } }

{ id: 'talk_minjae_notes',
  description: '"노트 빌려줄까? 어차피 내가 다시 정리할 거니까"',
  effects: { intimacy: 2, academic: 1, mental: 1 } }

{ id: 'talk_yuna_song',
  description: '"이 노래 들어봐, 너 좋아할 것 같아" 이어폰 한쪽 건네기',
  effects: { intimacy: 2, talent: 1, mental: 1 } }

{ id: 'talk_haeun_quiet',
  description: '"...괜찮아?" 자판기 옆에서 콜라 건네기, 침묵',
  effects: { intimacy: 2, mental: 2 } }

{ id: 'talk_junha_riceball',
  description: '"이거 어제 만들어봤어, 한번 먹어봐" 도시락통 내밀기',
  effects: { intimacy: 2, mental: 1, fatigue: -3 } }
```

50 단계는 이 결을 한 발짝 더 깊게 — **"가벼운 호의"가 아니라 "두 사람만의 코드"**.

### MiniTalkEvent 타입 시그니처

```typescript
export interface MiniTalkEvent {
  id: string;                            // 'talk_<npcId>_50_<keyword>' 형식
  npcId?: string;                        // NPC 이벤트
  intimacyMin?: number;                  // 친밀도 하한 (50으로 설정)
  gender?: 'male' | 'female';            // 성별 분기 시만
  description: string;                   // 본문 (NPC 대사 + 상황, 1~3줄)
  effects: {
    intimacy?: number;                   // NPC 친밀도 변화 (50 단계는 +3)
    stats?: Partial<Stats>;              // academic / social / talent / mental / health (creativity 없음)
    fatigue?: number;                    // 피로 ±1~3
    money?: number;                      // 돈 ±1~2 (있을 수도 없을 수도)
  };
  message: string;                       // 효과 한 줄 요약 (UI 노출)
}
```

### 효과 가이드라인 (친밀도 50 단계)

| 항목 | 권장 범위 |
|---|---|
| `intimacy` | **+3** (고정) |
| `stats.*` | ±1 (단일 스탯 또는 2개 스탯, 합 ±1~2) |
| `fatigue` | ±1 (0이어도 OK) |
| `money` | ±1~2 (있을 수도 없을 수도) |
| `memorySlotDraft` | **사용 금지** (Phase 2.4=친밀도 90 영역) |

### 작성 체크리스트

각 시드 작성 시 다음을 확인:

- [ ] **그 NPC만 할 수 있는 일인가?** (다른 NPC로 바꿔도 통하면 빈약 — 다시 쓸 것)
- [ ] **50 단계 톤("두 사람만의 코드")에 맞는가?** 30 단계의 가벼운 호의 수준이거나 70 단계의 속마음 수준이면 X
- [ ] **선택지가 없는가?** (description + effects + message만)
- [ ] **description이 1~3줄로 한 화면에 들어가는가?**
- [ ] **학년 종속 표현이 없는가?** (junha는 예외 — 고등 톤 OK)
- [ ] **다른 NPC가 주연으로 끼어들지 않는가?**
- [ ] **기존 30 단계 시드와 컨셉 중복이 없는가?**

### 채택 후 적용 위치

`game/src/engine/talkSystem.ts`의 `NPC_MINI_EVENTS` 배열에 신규 객체 6개 append.

신규 시드 ID 작명 규칙:
- `talk_<npcId>_<intimacyTier>_<keyword>` (예: `talk_jihun_50_pcbang`, `talk_yuna_50_lyric`)
- intimacyTier는 30/50/70/90 중 하나
- 임계값을 ID에 박는 이유: 풀 단계 구분이 ID에서 즉시 보이게 (Phase 2.3/2.4와 구분)

### 검증 절차

1. AI 결과 3~4개 모아서 좋은 시드 채택 (비교 표 만들기 권장)
2. `talkSystem.ts`에 객체 추가
3. `npx tsc --noEmit -p tsconfig.app.json` — TS 타입 검증 통과
4. `npx tsx scripts/verify-followup-chain.ts` — 회귀 없음
5. `npx tsx scripts/sim-y1-qa-matrix.ts` — 360 시나리오 0 실패
6. 인게임: 친밀도 50 도달 → 잡담 클릭 → 새 미니 이벤트 발동 확인 (NPC 6명 각각)
7. talkEventsFired 기록 확인 (1회 발동 후 풀에서 제외되는지)
