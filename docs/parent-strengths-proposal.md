## 0. 핵심 원칙 (3-way 피드백 합의점)

> **부모 = 선택 편향 + 사건 개입 + 회상 차별화.** 단순 수치 보너스 아님.

세 피드백이 공통으로 강조한 것:
- **Cursor**: 구현도 25% (완전 1 / 부분 1 / flavor 3 / noop 1), 조합 상호작용 0건. **축을 먼저 선명하게**.
- **Gemini**: `types.ts` 통합 스키마·M2→M5 단계 로드맵·`info` 예측 UI는 좋은 UX 포인트.
- **GPT (사고모델)**: 수치 중심이면 "빌드 싸움"으로 수렴. **러닝 + 선택 편향 + 기억 태그 3층이 한 묶음**으로 가야 서사가 된다.

## 1. 현재 상태 (Cursor 진단)

| 타입 | 초기 스탯 | 러닝 효과 | 활동 잠금 | 대사 분기 | 실구현 |
|------|-----------|-----------|-----------|-----------|--------|
| `emotional` | mental+10 | 매주 피로 +1~2 | 2개 | 있음 | **완전** |
| `wealth` | 없음 | 주당 순수입 +3.7만 | 없음 | 없음 | **부분** |
| `gene` | academic+5, health+5 | 없음 | 없음 | 없음 | **flavor** |
| `strict` | academic+5, mental-3 | 없음 | 없음 | 몇 줄 | **flavor** |
| `freedom` | 없음 | 방학 슬롯+1, 죽은 코드 | 없음 | 없음 | **거의 noop** |
| `info` | 없음 | 없음 | 없음 | 없음 | **noop** |

**구현도: ≈ 25%. 조합 전용 상호작용: 0건.**

## 2. 3층 구조 설계 (GPT 사고모델 핵심)

각 부모는 **3개 층위**가 동시에 발동해야 "서사 필터"로 작동.

| 층 | 역할 | M5 | M6 |
|---|---|---|---|
| **① 러닝 효과** | 숫자 차이 (엔진 루프에서 읽힘) | ✅ 6타입 전부 | (M5 완료) |
| **② 가시화** | WeekLog 1줄 "부모 보너스", 체감·버그 제보 동시 확보 | ✅ | (확장) |
| **③ 기억 태그** | `memorySlots` 카테고리 가중치 — 엔딩 회상 변화 | ✅ 경량 (가중치 테이블) | (심화) |
| **④ 선택 편향** | 이벤트에서 부모별 추가/잠금 선택지 | ❌ | ✅ 3부모만 (emotional/wealth/strict) |
| **⑤ 조합 시너지** | 15조합 중 3~4개 특수화 | ❌ | ❌ (M7+) |

**핵심:** M5 최소안이 **①+②+③을 동시에** 담으면, GPT가 경고한 "수치 빌드화"로 수렴하지 않음. Cursor의 "축 선명화"와 GPT의 "서사 필터" 요구를 동시에 만족.

## 3. 타입별 설계 (3층 통합)

### 3.1 `emotional` — 정서 방파제 (유지+강화)

- **러닝**: 매주 피로 -1~2 회복 (현행 유지)
- **가시화**: "엄마가 저녁에 물어봐줬다" 같은 WeekLog 1줄
- **기억 태그**: `reconciliation`, `growth` 가중치 +0.5

### 3.2 `wealth` — 자본가 (GPT 재정의 채택)

- **러닝**: 주당 +3.7만원 (유지)
- **가시화**: "학원비 걱정 없음" WeekLog 1줄
- **기억 태그**: 우회/보상 계열 가중
- **(M6) 선택 편향**: 위기/갈등 이벤트에서 "돈으로 해결" 선택지 활성화 (예: 시험 위기 → 과외 투입 / NPC 갈등 → 선물 봉합)
- **그림자**: 친구와 거리감 낼 수 있는 서사 트리거 (M6)

### 3.3 `info` — 한 발 먼저 아는 집 (Gemini 예측 UI + GPT 재정의)

- **러닝 M5 (A안)**: 학원/자습 계열 효율 +10%
- **가시화**: "엄마가 족보 구해왔다" WeekLog
- **기억 태그**: 준비/예측 계열 가중
- **M6 (B안)**: 시험 예측 UI — 시험 주간 전 예상 등급·부족 과목 툴팁

### 3.4 `strict` — 원칙주의자 (GPT 재정의: 제약과 안정)

- **러닝**: 루틴 보너스 도달 -1주 (3→2, 6→5, 8→7)
- **가시화**: "정해진 시간에 책상" WeekLog
- **기억 태그**: `failure`, `growth` 가중 (압박 후 성장 서사)
- **(M6) 선택 편향**: 자유/즉흥 선택지 일부 불리 또는 텍스트 변경 (리셋 코스트 ↑ 등)

### 3.5 `gene` — 체력/회복탄력성 (GPT 지적: 네이밍·밸런스)

- **러닝**: 피로 증가 -15%, 체력/운동 효율 +10%
- **가시화**: "오늘도 거뜬히" WeekLog
- **기억 태그**: `growth` 가중
- **⚠️ 결정 필요**: 네이밍 이슈 — "gene"은 현대적으로 민감할 수 있음. `resilience`/`basics`/`stamina` 등 중립적 축으로 **리네임 여부** 의사결정자 판단 필요.

### 3.6 `freedom` — 즉흥형 (기존 설계 유지+강화)

- **러닝**: `idleWeeks` 페널티 -50%, 루틴 리셋 자유, **죽은 코드 정리**
- **가시화**: "알아서 해" WeekLog
- **기억 태그**: `discovery` 가중
- **예측 불가능성**: 랜덤 이벤트 풀 가중치 약간 ↑ (M6)

### Q4. `gene` 부모 리네임 여부

**선택지**
- □ A. 유지 — `gene`
- □ **B. 리네임 — `resilience`** (회복탄력성) — 중립적, 서사적
- □ C. 리네임 — `basics` (기초 체력)

**권장**: **B. `resilience`**. "회복탄력성"이 본래 의도(피로 -15%·체력+10%)와 잘 맞고, 중립적.

## §9. 도메인 전문가 4명 피드백 (2026-04-24)

Q9 결정에 따라 시스템·서사·UX·밸런스 각 분야 수석급 전문가 4명 병렬 소환, 본 문서 v2를 검토받음.

### §9-A. 시스템 디자이너 — 판정: **보완 필요**

**핵심 반영 항목** (§4 Phase 1에 반영 완료)

1. **`parentModifiers.ts` 단일 객체 강제** — 현재 분기가 7개 이상 파일·15개소에 흩어져 있고, M6 선택편향 추가 시 50+로 폭발 위험. 신규 파일 `game/src/engine/parentModifiers.ts`에 `getParentMods(parents)` 셀렉터 두고 각 시스템이 이것만 참조.
2. **`strict` ↔ `emotional` 직교화 (시간 프레임 분리)** — strict는 `routineWeeks`만 건드림(멘탈/피로 직접 개입 금지), emotional은 `fatigue/mental` 직접 회복만(루틴 주수 개입 금지).
3. **6부모 모두 `parentFlavorText` 이벤트 톤 변형** — M6에 3부모만 선택편향 넣으면 나머지 3부모는 "스탯 덩어리" 인상. `GameEvent.parentFlavorText?: Partial<Record<ParentStrength, string>>` 1필드만 추가해서 6부모 모두 서사 목소리 부여.
4. **기억 bias를 tiebreaker로 강등** — 곱셈 연산(1.5×)은 초반 편향을 굳힘. `importance` 동률 구간에서만 +0.3 이하 가산점. `phaseTag` 쿼터(early/mid/late 최소 2개)로 학년 편향 차단.
5. **WeekLog에 `toneTag` 미리보기** — "엄마가 물어봐줬다 · 화해로 남을지도" 식으로 기억 슬롯 후보임을 예고 → ①러닝과 ③기억의 시간축 밀도 차이 해결.

### §9-B. 서사 디렉터 — 판정: **`wealth` 그림자 비어있음 최우선**

**핵심 반영 항목**

1. **`gene` UI 라벨 = "체질"** — 타입명 `resilience` 유지, UI 표기만 "체질" (한국 2013년 톤에 맞는 "타고난 쪽" 뉘앙스 보존).
2. **`wealth` 신규 기억 카테고리 2개** (부록 B 공백 채움):
   - `bypass` (우회) — "학원비로 건너뛴 순간"
   - `unspoken_debt` (갚을 수 없는 감사) — "책상 위 말없이 놓인 흰 봉투"
   - 가중치: `reconciliation` 0.8로 **낮춤** (돈이 관계 갈등을 "풀지 않고 덮는" 구조).
3. **`wealth` 그림자 이벤트 3개** (M6):
   - [중1 겨울] "너네 스키장 갔다 왔어?" — 수빈이가 "우리 아빠는 그런 거 안 해줘" 침묵
   - [중3] 학원 레벨 갈라짐 — "지훈이가 학원 얘기를 먼저 꺼내지 않게 됐다"
   - [고2] 생일 선물 역전 — 민재가 편지만 내밀 때 `wealth` 전용 "비싼 답례"(→"부담스러웠어" 해금)
4. **`strict` 부모 대사 3줄** (M6):
   - "너 지금 몇 시야. 내일 학교 어떻게 가려고." (야간 외출 시 리셋 코스트 +1)
   - "그거 해서 뭐가 되는데. 일단 이번 모의고사부터 보고 얘기해."
   - "엄마가 너 싫어서 그러는 거 아니야. 나중에 고맙다고 할 거야." (루틴 7주차 보너스 직전 WeekLog)
5. **6부모 엔딩 회상 초안** (20~35자, 시각 앵커):
   - emotional: *엄마가 현관에서 기다리던 노란 불빛.*
   - wealth: *책상 위, 말없이 놓여있던 흰 봉투.*
   - info: *식탁에 펼쳐진, 빨갛게 밑줄 쳐진 신문.*
   - strict: *11시, 스탠드를 끄러 오던 슬리퍼 소리.*
   - resilience: *감기에도 멀쩡하게 들고 가던 가방끈.*
   - freedom: *"알아서 해" 뒤에 닫히던 안방 문.*

### §9-C. UX 디자이너 — 판정: **WeekLog 1줄만으론 상기 실패**

**핵심 반영 항목**

1. **활동 결과 카드 인라인 부모 스티커 (M5 최우선)** — WeekLog 1줄보다 효과 큼. "`집중 공부 +12 📘 🎓+2`" 식으로 **숫자 옆에 부모 아이콘 + 1회성 펄스**. 맥락 근접성이 로그보다 월등.
2. **상단 HUD 부모 칩 2개** — 22×22 원형, 우측 인라인 정렬 (별도 줄 금지). 발동 시 0.5초 `box-shadow` 펄스만, 상시 발광은 피로감.
3. **3중 앵커 구조** — 활동 결과 인라인(주 앵커) + HUD 아이콘(상시) + 엔딩 "흔적"(회수) — WeekLog 1줄은 보조 채널로 격하.
4. **`info` 예측 UI (M6)** — 시험 주 전 **주말 루틴 편성 화면 우측 상단 "엄마 메모" 접힌 카드**. 시험 당일 직전 금지(스포일러). 구간 표시(B+~A-), 1과목만 노출("국어가 좀 불안하대"), 난이도 ±1등급 랜덤 유지.
5. **엔딩 "부모 흔적"** — 엔딩 스크린 내 통합 권장 (별도 챕터 2 금지). NPC 에필로그 직후, plum 배경 카드 + 부모 아이콘 대형 + 최대 3줄.
6. **부모 전용 선택지 시각화 (M6)** — 배경 틴트 + 좌측 아이콘 바 + 속삭임 한 줄("엄마라면 이러라고 했을 거야").

   - `strict+info` — 루틴+학업 실효 +40% 성장
   - `wealth+info` — 진로 3노선 동시 오픈, 엔딩 다양성 축소
3. **`freedom` 랜덤 이벤트 +20% 가중** M5 편입 — 부록에만 있던 장치를 본안에 반영. 축을 3.5개에서 4개로 끌어올림.
4. **리플레이 각도 부족** — "너는 어떻게 했어?" 대화 발생률 현재 20~30%. **부모별 고유 엔딩 최종 문장 1줄 분기** 필요(SNS 스샷 트리거). 예: strict는 "규칙이 날 지켜줬다", freedom은 "아무도 날 안 막았다".
