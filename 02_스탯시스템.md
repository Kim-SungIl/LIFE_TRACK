# 02. 스탯 시스템 — 수치 설계서

> **이 문서는 현재 구현(`game/src/engine/gameEngine.ts`, `activities.ts`, `types.ts`) 기준이다.**
> 초기 설계(2026-04~05)의 정교한 보정 체계(체력/멘탈 8단계 성장보정, 각성 상태, 하위스킬, 부모 안전망 등)
> 상당수가 밸런스 sim·QA 반복을 거치며 단순화되거나 미구현 상태다. 미구현/보류는 §12에 격리했다.
>
> 설계 원칙(유지):
> - 7년(336주) 동안 스탯이 의미있게 성장하되, 만렙은 사실상 불가능
> - "모든 스탯 올리기"는 구조적으로 불가능 (핵심 철학: 포기의 고통)
> - 수치는 내부적으로 작동하되, 플레이어에게는 단순하게(5스탯+등급) 보여야 함

## 1. 스탯

### 1-1. 보이는 스탯 (5종)

| 스탯 | 범위 | 설명 |
|------|------|------|
| 학업 (academic) | 0~100 | 학습 능력, 시험 성적에 직결 |
| 인기 (social) | 0~100 | 교우관계, 사회성 |
| 재능 (talent) | 0~100 | 예체능·특기 숙련도 |
| 멘탈 (mental) | 0~100 | 정신적 안정 |
| 체력 (health) | 0~100 | 신체 건강·컨디션 |

**플레이어 표시**: 수치 대신 5단계 등급(`getGrade`, `STAT_GRADES`). E(매우 부족) / D(부족) / C(보통) / B(우수) / A(최상).
스탯별 짧은 의미 라벨(`STAT_FLAVOR_LABELS`)도 제공 — 예: 학업 A "상위권", 인기 E "외톨이".

### 1-2. 시작값 (`createInitialState`)

| 스탯 | 기본값 |
|------|--------|
| 학업 | 30 |
| 인기 | 25 |
| 재능 | 15 |
| 멘탈 | 50 |
| 체력 | 40 |

- 멘탈은 "깎이는 자원" 성격이라 중간값(50)에서 시작, 체력은 운동 가치를 위해 낮게(40).
- **부모 보정**(03_부모시스템 SSOT, `parentModifiers`): 초기 보정은 academic/mental/health에만 적용 — resilience(학업+5·체력+5), strict(학업+5·멘탈-3), emotional(멘탈+10). social/talent 초기 보정 없음.

### 1-3. 구간 감쇠 (`getDiminishingReturn`) — 양수 성장에만 적용

| 현재 스탯 | 배율 |
|-----------|------|
| 0~29 | ×1.2 (초반 가속) |
| 30~49 | ×1.0 |
| 50~69 | ×0.8 |
| 70~84 | ×0.5 |
| 85~94 | ×0.3 |
| 95~100 | ×0.1 |

> 30→60은 비교적 쉽지만 80→90은 극도로 어렵다. 한 스탯에 7년을 쏟아도 95+는 매우 힘들다.

## 2. 주간 행동 구조

학기 중 한 주의 행동은 **방과후 루틴 슬롯 + 주말 선택**으로 구성된다.

- **루틴 슬롯**: `routineSlot2`, `routineSlot3` — 방과후 고정 루틴 2개. 같은 활동을 일정 주(`routineSlotNWeeks`) 유지하면 루틴 보너스(§6) 발동.
- **주말 선택**(`weekendChoices`): 주말 활동 선택.
- **방학**(`vacationChoices`): 학기 대신 방학 전용 활동. freedom 부모는 방학 슬롯 +1.
- 이벤트가 발생한 주는 `eventTimeCost`만큼 슬롯이 차감된다.

> 초기 설계의 "주당 5슬롯 고정 + 시기별 강제 슬롯 표 + 자립도 기반 강제슬롯 저항"은 현 구현과 다르다(§12 보류).

## 3. 피로 시스템

### 3-1. 기본

- 피로(fatigue) 0~100, 시작 0. 누적 방식(무리하면 다음 주로 이월).

### 3-2. 주간 자연 회복 (`applyFatigueRecovery`, processWeek 1단계)

```
회복량 = max(3, min(12, fatigue × 0.15)) × mult
  - mult: 도전 모드(useReducedRecovery)면 0.6, 아니면 1.0
  - 정서적 지지(emotional 부모): +2 (단 멘탈 70+ 시 +1) × mult
  - 방학 중: +2 × mult
```

| 현재 피로 | 기본 회복(±보너스 전) |
|-----------|----------|
| 0~19 | -3 (최소 보장) |
| 40 | -6 |
| 60 | -9 |
| 80+ | -12 (최대) |

### 3-3. 피로 보정 (`getFatigueModifier`) — 활동 성장 효율에 곱셈

| 피로 구간 | 배율 |
|-----------|------|
| 0~19 | ×1.0 |
| 20~39 | ×0.9 |
| 40~59 | ×0.75 |
| 60~79 | ×0.5 |
| 80~89 | ×0.3 |
| 90~100 | ×0.2 |

### 3-4. 학교 수업 (`applySchoolClass`, 학기 중 평일 자동)

- 학업 += `0.3 × getDiminishingReturn(academic) × getFatigueModifier(fatigue)` (초기 설계 +1에서 대폭 하향).
- 피로 += 2.

### 3-5. 활동 피로 비용 보정 (`applyActivity` 내 피로 적용)

활동의 기본 피로 증가량(`activity.fatigue`)은 아래 순서로 보정된다(양수일 때):
1. 멘탈 상태: burnout ×0.5 / tired ×0.75 (지친 상태에선 활동 강도 체감 감소 — tired 함정 방지).
2. **체력 기반 피로 저항**: 체력≥60 -2 / 체력≥30 -1 (최소 1 보장).
3. 체력<20: ×1.5 (허약 → 더 쉽게 지침).
4. resilience 부모: ×0.85 (체질 — 피로 증가 -15%).

> 즉 "체력→피로 내성"은 8단계 정밀표는 아니어도 **간략 형태로 구현**돼 있다(피로 증가량 한정. 회복량 보정은 없음).

## 4. 멘탈 상태 시스템 (`checkMentalStateTransition`)

### 4-1. 3단계 상태

```
normal(정상) ←→ tired(피로) ←→ burnout(번아웃)
```
> 초기 설계의 4번째 "각성(覺醒)" 상태는 미구현(§12). mentalState 타입은 normal/tired/burnout 3종.

### 4-2. 전환 게이트

| 전환 | 조건 |
|------|------|
| normal → tired | (mental<40 & fatigue>45) **또는** fatigue≥85 |
| tired → normal | mental≥42 & fatigue<55 (2점 히스테리시스) |
| tired → burnout | mental<20 **또는** (mental<25 & fatigue>70) — 단 번아웃 쿨다운 중이면 차단(대신 mental+2 면역) |
| burnout → tired | mental≥20 & fatigue<40 → 이후 8주 번아웃 쿨다운(면역) |
| **강제 burnout** | tired 24주+ 연속 & 쿨다운0 → mental 무관 강제 전환(갈아넣기의 서사적 대가) |

> `burnoutCount`는 **burnout 진입 시에만 1회** 증가(일반 진입 + 강제 진입). 주간 회복 처리(§4-3)에선 증가하지 않는다.

### 4-3. 상태별 자동 처리 (매주)

- **tired**: 피로 -5(도전 -3). 단 fatigue<60이면 mental+2. 8주+ 연속 & fatigue<60이면 자력 탈출 보조(mental+1.5, fatigue-3).
- **burnout**: 피로 -12(도전 -8), mental+4(도전 +3). (burnoutCount는 여기서 증가하지 않음 — §4-2 진입 시 1회.)
- 누적 카운터: `consecutiveTiredWeeks`(연속), `totalTiredWeeks`(누적 — 엔딩 만성탈진 라우팅용).

### 4-4. 상태별 활동 성장 패널티 (`applyActivity`의 mentalPenalty)

- burnout: ×0.35
- tired: ×0.8 (만성 — consecutiveTiredWeeks≥16 ×0.5, ≥10 ×0.75)
- normal: ×1.0

### 4-5. 멘탈 회복 감쇠 (`getMentalRecoveryRate`) — 멘탈 회복 활동에 곱셈

| 현재 멘탈 | 배율 |
|-----------|------|
| 0~69 | ×1.0 |
| 70~79 | ×0.8 |
| 80~89 | ×0.55 |
| 90~100 | ×0.3 |

## 5. 자연 감소 (`applyNaturalDecay`, processWeek 5단계)

| 스탯 | 감소량 | 하한 |
|------|--------|------|
| 학업 | 학년별: 초(Y1) 학기 -0.1/방학 -0.3 · 중(Y2~4) -0.15/-0.4 · 고(Y5~7) -0.3/-0.7. 95+ 추가 -0.5 | 12 |
| 인기 | -0.8 (≤12이면 -0.2 완화 / 학기 중 <30이면 +0.2 미세회복) | 0 |
| 재능 | -0.2 | 5 |
| 체력 | 방학 -0.8 / 학기 -0.4 (≤12이면 -0.1) | 10 |
| 멘탈 | 80~89 -1 / 90~100 -2 (그 외 자연감소 없음) | — |

**추가 멘탈/체력 침식 (같은 함수 내)**:
- 고피로 → 멘탈: fatigue 80+ -4 / 60+ -2 / 40+ -1.
- 인기<20 → 멘탈 -1.5 (사회적 고립).
- fatigue 90+ → 체력 -1.

> 초기 설계의 "비활동 80+ 스탯 추가 감소(§13)"는 미구현(§12). 멘탈 자연감소(80+/90+)는 구현됨.

## 6. 성장 공식 (`applyActivity`)

양수 성장 스탯(멘탈 제외):
```
value = base
      × getDiminishingReturn(현재 스탯)      // 구간 감쇠
      × getFatigueModifier(현재 피로)         // 피로 보정
      × mentalPenalty                         // 멘탈 상태(normal/tired/burnout) 패널티
      × (1 + buffBonus + routineBonus + parentEfficiencyBonus)  // 가산 버프
      × efficiency
```
- **멘탈**은 별도: `base × getMentalRecoveryRate(mental)`. 무기력 3주+(idleWeeks≥3)에 휴식 계열이면 추가 ×0.3.
- **중복 효율 감소**(같은 주 같은 스탯): 직전 누적>2면 ×0.45, >0.5면 ×0.7.
- **무료 활동 soft cap**: 비용 0 활동 & 스탯 80+면 ×0.1.
- **최저 보장**: 스탯<85 & 결과<base×0.1이면 base×0.1로 끌어올림.
- **주당 상한**: 멘탈 외 스탯은 +2/주까지만 누적. (단 방학 catchup 보너스는 이 상한 뒤에 가산되어 +2를 넘을 수 있음.)

**루틴 보너스 (`getRoutineBonus`)** — 같은 루틴 유지 주수(12주 캡):
- 8주+ ×0.3 / 6주+ ×0.2 / 3주+ ×0.1 (가산).

**부모 활동 효율 (`parentModifiers`)**: info 학업 +0.1 / resilience 운동 +0.1.

스탯이 임계값에 처음 도달하면 1회 메시지가 표시된다(성장 체감). **현재는 메시지 출력만 — 보너스 효과는 미구현**(§12).

| 스탯 | 임계값 |
|------|--------|
| 학업 | 40 / 55 / 70 / 85 |
| 인기 | 35 / 50 / 70 |
| 재능 | 30 / 50 / 70 |
| 체력 | 50 / 70 |
| 멘탈 | 70 |

> 초기 설계의 "전 스탯 30/50/70/85 통일 + 마일스톤별 보너스 효과(부모관계+10, 용돈+2 등)"는 위 임계값/효과 모두 미반영.

## 9. 활동 테이블

**활동 수치(슬롯/피로/스탯/용돈)의 SSOT는 `game/src/engine/activities.ts`다.** 설계 문서는 계열·설계 의도만 기술하고, 구체 수치는 코드를 따른다(드리프트 방지).

계열 구성(대표 예시):
- **공부**: 독학·학원·인강·스터디그룹·도서관자습·EBS 등. 학원은 고효율·고비용·고피로, 무료 활동은 80+ soft cap.
- **운동**: 가벼운 운동·체육부·헬스 등. 체력↑, 피로 관리.
- **관계**: 친구와 놀기·SNS·동아리·데이트 등. 인기·멘탈↑.
- **자기계발**: 예체능 레슨·코딩·창작·봉사 등. 재능↑.
- **휴식**: 휴식·푹 쉬기·게임/영상. 피로↓, 성장 없음.
- **알바**: 편의점·과외(받기/하기)·배달 등. 용돈↑.
- **부모 전용/방학 전용**: emotional 가정 활동(부모와 공부·가족 저녁), 방학 활동(도서관·시골·가족여행·스포츠캠프 등 9종, `seasonGate: 'vacation-only'`).

설계 의도(유지):
- **저소득 = 하드모드가 아니라 안정형 빌드**. 무료 활동(도서관·EBS·동네체육관)으로 중상위권까지 가능하되, 최상위권(85+)은 양질의 교육 자원(학원·과외) 필요 — 무료 활동 80+ soft cap이 이를 구현.
- 휴식은 피로를 크게 줄이지만 다른 스탯이 안 큰다 → "쉬어야 하는데 쉴 수 없는" 딜레마.

## 10. 주간 처리 순서 (`processWeek`)

1. 피로 자연 회복 → 2. 학교 수업(학기 중만) → 3. 루틴 활동 → 4. 주말/방학 활동 →
5b. 부모 친밀도 평균 회귀 → 5. 자연 감소 → 6. NPC 친밀도 감소 →
**7. 멘탈 상태 전환 → 8. 마일스톤** → 9. 용돈/생활비 → 10. 방치 무기력 →
스탯 범위 보정 → 시험 체크 → 이벤트 체크 → 버프 틱다운 → 주 진행.

> 멘탈 전환(7)이 활동 적용(2~4) 뒤·마일스톤(8) 앞에 오는 순서가 중요.
> (앞단에 NPC 동행 친밀도 적용·주 컨텍스트 준비, 후단에 주말/방학 선택 초기화가 추가로 있다.)

## 12. 미구현 / 보류 (초기 설계 중 현재 코드에 없는 것)

**미구현 (현재 코드에 없음)**
- **각성(覺醒) 상태** — mentalState 4번째 단계 및 트리거 스탯 ×2.0 성장 등(§초기 4-2~4-4).
- **하위 스킬 체계** — 문과/이과 지수, 음악/미술/체육, 친화력/존재감 등. 현재는 `Track`(humanities/science, Y6 선택)만 구현. 내부 세부스탯(평판 태그·자립도 등) 대부분 미구현.
- **체력/멘탈 8단계 성장 보정** — 초기 설계의 체력→성장·멘탈→성장 8단계 곱셈 보정표. 현 구현은 멘탈 상태 패널티(mentalPenalty) + 멘탈 회복 감쇠(getMentalRecoveryRate)만. (체력→**성장** 보정은 없음 — 단 체력→**피로** 저항은 §3-5로 구현됨.)
- **부모 개입 안전망(Pity System)** — 피로85/멘탈15 시 부모 유형별 강제 개입. (fatigue≥85는 tired 진입 조건일 뿐, 강제 개입 시스템은 없음.)
- **비활동 80+ 스탯 추가 감소** — 80+ 스탯 미활동 시 -0.5/-1.0 추가. (단 idleWeeks 방치 페널티 — 3주+ rest-only면 멘탈/소셜 감소, freedom 절반 — 는 구현됨, §10 방치 무기력.)
- **마일스톤 보너스 효과** — 현재는 메시지만, 부모관계+10·용돈+2 등 효과 없음.
- **멘탈 마일스톤 50(8주 유지)·85**, 재능 85, 체력 30 등 일부 마일스톤 임계.

**형태가 다르게 구현됨 (초기 설계와 다른 방식으로 존재)**
- **컨디션 난조(`sick-day`) 이벤트**: 초기 설계의 "체력 25 이하 확률 병결·슬롯 손실"과 달리, **fatigue≥40 & 학기 중** 트리거되는 학교생활 랜덤 이벤트로 존재(참고 등교 / 하루 쉬기 2선택지). 슬롯 손실·체력 게이트 형태는 아님. (school-life.ts)

**구식 표기 (초기 문서에 있던 구 수치)**
- 자연회복 "-8 고정", 자연감소 "학업 -0.5/인기 -1.0/재능 -0.3" 등 — 현 코드값은 §3·§5 참조.
- 피로 단계 7구간(20~34/35~49…) — 현 getFatigueModifier는 6구간.
- 멘탈 전환 게이트 "멘탈<30/<15" — 현 코드는 "<40&피로>45 / <20".
