# UI/UX 방향성 — 디렉터 판단 요청

> 프로토타입 개발이 완료되었습니다. 핵심 시스템(스탯, 활동, 이벤트, NPC)이 작동하는 것을 확인했습니다.
> 이제 "게임처럼 느껴지는 UI"를 입혀야 하는 단계입니다.
> GPT 디렉터의 사전 피드백이 포함되어 있습니다. 교차 검증 및 추가 의견을 부탁드립니다.

---

## 현재 프로토타입 상태

- Vite + React + TypeScript 웹 게임
- 기본 시스템 작동: 5스탯, 활동 22종, 이벤트 15종, NPC 4명, 성별 선택
- AI 생성 캐릭터 일러스트 적용 (남녀 주인공 + NPC 4명)
- UI: 관리 시뮬레이션 스타일 (스탯 바 + 카드 선택 그리드)

### 현재 UI의 문제점
- 텍스트와 숫자 위주 → "게임"보다 "대시보드/학습관리 앱" 느낌
- 이벤트가 텍스트 박스 안에서만 진행 → 몰입감 부족
- 배경 일러스트 없음 → 장소감이 없음
- 캐릭터가 작은 아바타로만 등장 → 감정 전달 약함
- 활동 카드가 숫자(학업+2, 피로+6)만 → 전략적 재미는 있지만 서사적 재미 부족

## GPT 디렉터의 사전 피드백 요약

### 추천 방향: "학생 플래너 + 감성 육성 시뮬레이션" 혼합형

> UI 구조는 **플래너/다이어리처럼 이해하기 쉽게**,
> 감정과 분위기는 **청춘 육성 게임처럼**,
> 선택지는 **이벤트 카드처럼**.

### GPT가 제시한 핵심 개선 5가지

**1. 시간 흐름 연출**
- 달력/학기 진행 바
- 주차 전환 시 짧은 연출
- 계절에 따른 배경 톤 변화
- 시험기간/축제/방학 시즌 배너

**2. 캐릭터 감정/상태 연출**
- 표정이 피로/멘탈에 따라 변화 (이미 부분 구현)
- 말풍선 한 줄: "이번 주는 좀 할 만한데?", "숙제가 너무 많아..."
- 활동 선택 시 캐릭터 반응
- 주말 행동 확정 전 짧은 코멘트

**3. 활동 카드에 서사 요소 추가**
- 숫자(학업+2)만이 아니라 설명문 + 분위기 태그
- 예: `독학` → "조용히 책상에 앉아 진도를 뺍니다." + `집중`, `혼자`, `지루할 수 있음`
- 예: `스터디 그룹` → "친구들과 같이 공부하며 친밀도도 오른다." + `친구`, `협동`, `변수 있음`
- 추천 상황 태그: `피로 낮을 때 추천`, `시험 전 유리`, `가성비 좋음`

**4. 스탯 게임화**
- 아이콘 부여 (학업:📚, 인기:⭐, 재능:💡, 멘탈:🍀, 체력:⚡)
- 등급을 텍스트 라벨로: `평범`, `좋음`, `뛰어남`
- 수치 도달 시 작은 배지: "성실함", "인기 급상승"
- 증감 시 바가 짧게 튀는 모션

**5. 배경을 단색→장면형으로**
- 교실 창가, 집 책상, 운동장, 학원 복도 등
- UI를 해치지 않을 정도로 세계관을 암시
- 계절/시간대에 따라 변화

### GPT 추천 적용 순서

```
1단계: 프레임만 게임답게 (HUD, 배경, 아이콘, 카드 설명)
2단계: 캐릭터성 추가 (표정, 말풍선, 주간 요약, 알림 토스트)
3단계: 세계 확장 (관계탭, SNS탭, 계절/학년 변화 연출)
```

## 디렉터에게 묻는 질문

### 1. GPT의 "학생 플래너 + 감성 육성 시뮬레이션" 방향에 동의하는가?

GPT가 제시한 3가지 방향 중:
- 방향 1: "감성 육성 시뮬레이션" — 페르소나/프메 스타일
- 방향 2: "학생 플래너/다이어리" — 현재 구조와 가장 가까움
- 방향 3: "청춘 드라마 HUD" — 더 몰입적이지만 리소스 부담 큼

GPT는 1+2 혼합을 추천. 동의하시나요? 다른 방향이 있나요?

### 2. 비주얼 노벨 요소를 어디까지 넣어야 하는가?

현재 이벤트에만 NPC 일러스트+대화가 나오는데:
- **A.** 이벤트만 비주얼 노벨, 나머지는 관리 UI (현재 방향)
- **B.** 스케줄 선택도 비주얼 노벨 포맷 (주인공 독백 + 선택지)
- **C.** 모든 것을 비주얼 노벨로 (순수 VN)

어디까지가 적절한가?

### 3. 336주 반복의 템포 문제를 어떻게 해결할 것인가?

- 매주 같은 화면이면 질림
- 매주 다른 연출이면 리소스 폭발
- GPT는 "시즌 배너 + 계절 배경 변화"로 완급 조절 제안
- 추가 아이디어가 있는가?

### 4. "숫자를 보여줘야 하는가, 숨겨야 하는가?"

GPT는 "숫자 + 서사 조각을 같이 보여주라"고 했는데:
- 프로토타입 단계: 숫자 노출 OK (시스템 검증)
- 정식 출시: 숫자를 어디까지 숨기고 서사로 대체해야 하는가?
- "포기의 고통"은 숫자가 보여야 더 아픈가, 서사로 전달해야 더 아픈가?

### 5. 이 게임의 UI를 한마디로 정의한다면?

## Claude(시스템 담당) 의견

GPT의 피드백이 매우 구체적이고 실행 가능합니다. 특히:

1. **"학생 플래너" 방향**은 현재 프로토타입 구조를 크게 바꾸지 않고도 게임 느낌을 올릴 수 있어 효율적
2. **활동 카드에 서사 요소 추가**는 즉시 적용 가능하고 체감 차이가 클 것으로 예상
3. **적용 순서(프레임→캐릭터→세계)**는 리소스 관리 관점에서 합리적

다만 **비주얼 노벨 요소의 범위**에 대해서는 Gemini의 의견이 필요합니다.
이벤트만 VN으로 할지, 스케줄 선택까지 VN화할지에 따라 개발 방향이 크게 달라집니다.
