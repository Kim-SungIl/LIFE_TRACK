# 03. 부모(가정) 시스템 — 핵심 차별화 요소

> **이 문서는 현재 구현(`game/src/engine/parentModifiers.ts`, `parentIntimacy.ts`, `talkData/miniEvents.ts`, `talkSystem.ts`) 기준이다.**
> 초기 설계(2026-04)의 강점별 정교한 페널티/고유효과(성적압박·간섭감시·자유슬롯 보정·저축한도 등) 상당수가
> 밸런스 sim·QA를 거치며 단순화되거나 미구현이다. 대신 **부모 친밀도·미니이벤트·절정** 시스템이 새로 들어왔다.
> 미구현/보류는 §8에 격리했다.

## 설계 원칙

> "부모는 버프가 아니라 조건이다" · "부모 선택 = RPG의 클래스 선택"

모든 가정에는 강점과 약점이 공존한다. 사기적인 부모는 없다.

## 1. 강점 2개 선택

게임 시작 시 **가정의 강점 2가지**를 고른다(아빠/엄마 고정 아님 — 다양한 가정 형태 포용). 6개 중 2개 = 15조합.

| 강점 | 코드명 | 정체성 |
|------|--------|--------|
| 재력 | `wealth` | 자원(용돈)이 넉넉하다 |
| 정보력 | `info` | 학습을 효율적으로 안내한다 |
| 회복력 | `resilience` | 타고난 체질·멘탈 기반(초기 명칭 "유전자"에서 변경) |
| 정서적 지지 | `emotional` | 마음을 돌보는 안전한 집 |
| 시간/자유 | `freedom` | 간섭 않고 자율을 존중 |
| 엄격함/규율 | `strict` | 훈련·루틴으로 밀어붙임 |

> **용어 변경**: 초기 설계의 "유전자(외모/지능)"는 **`resilience`(회복력)**으로 리네임됐다.
> 구세이브는 `gene → resilience` 마이그레이션(`stateMigration.ts`)으로 호환.

## 2. 강점별 러닝 효과 (`parentModifiers.ts` SSOT)

부모 효과는 `getParentMods()` 한 곳에서 일괄 산출된다(gameEngine·examSystem 분기 없음). **두 강점의 효과는 합산**된다.

| 강점 | 초기 스탯 | 러닝 효과 |
|------|-----------|-----------|
| 재력(wealth) | — | 주당 용돈 **5만원**(그 외 강점 3만원) |
| 정보력(info) | — | 학업(study) 활동 효율 **+0.1** |
| 회복력(resilience) | 학업 **+5**, 체력 **+5** | 운동(exercise) 활동 효율 +0.1, 활동 피로 증가 **×0.85**(-15%) |
| 정서적 지지(emotional) | 멘탈 **+10** | 주간 피로 자연 회복 **+2**(단 멘탈 70+ 시 +1) |
| 시간/자유(freedom) | — | idle 방치 페널티 **×0.5**, 방학 슬롯 **+1** |
| 엄격함(strict) | 학업 **+5**, 멘탈 **-3** | 루틴 보너스 도달 **+1주 가속**(routineWeeksBoost). 시험 상위권 성적 향상(gradeImprove) 시 **연 1회 멘탈 +2**(B2, `parentPraiseYears`) |

- 초기 스탯 보정은 **academic/mental/health에만** 존재한다. social/talent 초기 보정은 어떤 강점에도 없다.
- 위 표가 강점 효과의 **전부**다. 초기 설계의 강점별 고유효과/페널티(성적압박·간섭·자유슬롯 보정·고유 천장 등)는 대부분 미구현(§8).
- `weeklyIncome`은 중복 시 합산이 아니라 **재력이면 5, 아니면 3**(정보력·엄격함 "4만원" 차등은 미구현, §8).
- 부모 효과 발동은 결산에 **부모 보너스 칩**(`parentBonusesApplied`)으로 가시화(HUD 펄스·인라인 칩·툴팁). 매주 동일 발동(wealth 용돈/resilience 체질)은 노이즈 차단 위해 결산 칩에서 제외.

## 3. 부모 친밀도 시스템 (`parentIntimacy.ts` SSOT)

> 설계: `docs/parent-intimacy-redesign.md` (v2). 부모 강점은 친밀도의 "자동 방향"이 아니라 **"행동 반응 배율"**이다.

- **parentIntimacy**: 0~100, **초기 50**. 모든 변화는 단일 진입점 `applyParentIntimacyDelta(state, baseDelta, tag)`를 통과한다(직접 가산 금지).

### 3-1. 행동 태그 (ParentTag, 13종)

shareWorry(고민공유) / hideProblem(숨김−) / familyTime(함께한 시간) / gradeImprove(성적향상) / gradeDrop(급락−) / keepPromise / breakPromise(−) / careerTalk(진로) / ignoreAdvice(조언무시−) / overspend(과소비−) / moneyRequest(용돈요청) / autonomyChoice(자기주도) / recoveryAction(만회).

### 3-2. 강점 × 태그 반응 배율 (`REACTION_TABLE`)

각 강점은 태그별로 **민감 1.4 / 기본 1.0 / 둔감 0.6** 배율을 가진다(강점 정체성). 예(전체는 코드 참조):
- emotional: shareWorry·hideProblem·familyTime·recoveryAction 1.4, gradeImprove·gradeDrop 0.6 (성적엔 둔감, 마음엔 민감).
- strict: gradeImprove·gradeDrop·keepPromise·breakPromise·recoveryAction 1.4, familyTime·autonomyChoice 0.6 (신뢰·성적·만회로 가까워짐).
- resilience: recoveryAction 1.4, gradeDrop·breakPromise·hideProblem 0.6 (하강에 둔감 — 기다려주는 부모).
- freedom: autonomyChoice·keepPromise 1.4, familyTime·gradeImprove 0.6 (자유=자기책임).

### 3-3. 적용 순서 (`applyParentIntimacyDelta`)

1. **두 강점 배율 곱** (0.36 ~ 1.96).
2. **사춘기 골짜기**(Phase 4D): Y3~Y4(중2~3)엔 familyTime/shareWorry 반응 ×0.75 — 이 시기 하락은 정상, 이후 화해가 의미를 갖도록.
3. **구간 감쇠**(연속): factor = 양수면 (100−pi)/50, 음수면 pi/50, **0.2~1.0 clamp**. (pi=50 ×1.0 / 70 ×0.6 / 85 ×0.3 / 100 clamp 0.2.)
4. clamp(0~100) 후 적용.
5. baseDelta>0이면 `parentPositiveTags[tag]` 누적(천장 감쇠로 적용량이 0에 가까워도 "의도" 기준 적립 — 절정 트리거의 "쌓을 이유").

### 3-4. 시험 반응 (`examParentEffect`)

직전 대비 석차 상승(중·고) → gradeImprove(+0.8). 그 외엔 mentalDelta 부호로 상위권→gradeImprove / 하위권·급락→gradeDrop(−1.0). 초등은 평균 75+→gradeImprove / 45 미만→gradeDrop.

### 3-5. 평균 회귀 (`applyParentMeanReversion`)

부모 관련 행동이 없는 주(`actedWithParentThisWeek=false`)엔 50으로 천천히 회귀. k = pi<50이면 0.006(회복 더딤) / pi≥50이면 0.01(반감기 ≈1.4년). "데면데면하면 평균."

### 3-6. 회고 톤 (`getParentIntimacyTone`)

<30 distant / 30~69 normal / ≥70 warm. 학년말 일기·엔딩 부모 에필로그(07_엔딩 §3-5) 톤에 사용.

## 4. 부모 미니이벤트 (`PARENT_MINI_EVENTS`)

집에서 발생하는 강점별 미니이벤트. **±선택지(긍정 vs 회피)**로 친밀도에 양방향 압력을 준다.

- 강점별 풀, `parentEffect: { baseDelta, tag }`로 §3 단일 진입점에 연결.
- **쿨다운 4주**(`PARENT_EVENT_COOLDOWN_WEEKS`) 후 재발동 — 긍정만 1회면 인플레되므로 반복 발동.
- 두 강점 이벤트가 번갈아 나오게 **로테이션**(가장 오래전 발동 우선).
- yearMin/yearMax 게이트, 선택지에 `memorySlotDraft`(회상 슬롯) 부착.
- 예: emotional "핫초코" — [털어놓기] shareWorry +1.5(reconciliation 슬롯) / [웃어넘기기] hideProblem −0.6(regret 슬롯).

## 5. 부모 절정 (`PARENT_CLIMAX_EVENTS`, Phase 4B)

> "이 부모라서 좋은 단 한 장면." 강점당 **평생 1회**(`parentClimaxFired` 가드). 선택지 없는 단일 컷.

### 5-1. 트리거 (`getEligibleParentClimax`)

복합 게이트 — 친밀도 단독 임계 금지(숨은 스탯 그라인딩 방지):
- **일반**: 친밀도 ≥ **60**(`INTIMACY_GATE`) AND 해당 강점 긍정 태그 누적 ≥ **2**(`TAG_THRESHOLD`) AND year ≥ climaxYearMin AND 미발동.
- **구제**: 고3(Y7) W36+(수능 후) 미발동분은 누적 무시, 친밀도 ≥ **45**(`RESCUE_GATE`)면 발동(놓침 방지). distant(<45)는 미발동이 서사.

### 5-2. 6개 절정

| 강점 | triggerTag | climaxYearMin | 화자 | 효과 |
|------|-----------|---------------|------|------|
| emotional | shareWorry | 2 | 엄마 | 회상 슬롯(importance 5) |
| resilience | recoveryAction | 3 | 엄마 | 회상 슬롯 |
| info | careerTalk | 4 | 엄마 | 회상 슬롯 |
| strict | gradeImprove | 4 | 아빠 | **멘탈 +2** + 회상 슬롯 |
| freedom | autonomyChoice | 4 | 엄마 | 회상 슬롯 |
| wealth | familyTime | 5 | 아빠 | 회상 슬롯 |

- **스탯 퍼주기 금지**: strict만 멘탈 +2 유지, 나머지 5축은 스탯 0 — 보상은 회상 슬롯(importance 5)이다.
- 화자 SSOT: 엄마(emotional/info/resilience/freedom) / 아빠(wealth/strict).
- 절정을 본 강점은 엔딩 부모 에필로그에서 warm 비트가 "그 순간 콜백(warmSeen)"으로 교체(07_엔딩 §3-5).

## 5b. 부모 갈등·화해 이벤트 (고정주차 풀 GameEvent)

미니이벤트(§4)·절정(§5)과 별개로, 부모 서사 아크가 고정 주차 이벤트로 들어가 있다. 진로 '결과'는 안 바꾸고(스탯 퍼주기 금지) 작은 멘탈·학업 + 친밀도 굴곡 + 회상만.

- **진로 갈등 (Phase 4C, `parent-career.ts`)**: "원하는 진로 vs 부모 기대" 줄다리기. **strict 아크(아빠, Y5 W15·Y6)** / **info 아크(엄마)**. 선택 3종 — 순응(keepPromise/careerTalk+) / 내 길(breakPromise/ignoreAdvice−) / 설득(autonomyChoice+). (info+strict 가정은 strict 우선.)
- **사춘기 골짜기 (Phase 4D, `parent-adolescence.ts`)**: 전 부모 공통(사춘기는 유형 안 가림). `adolescence-clash`(Y3 W26, "방문 쾅") → `adolescence-reconcile`(Y5 W26, 골짜기 겪고 친밀도≥60일 때만 해금, "그땐 미안했어"). §3-3 사춘기 배율 하락(×0.75)과 한 세트.

## 6. 정보력의 설계 철학 (보존)

> "아이의 입시 성공은 할아버지의 재력, 아빠의 무관심, 엄마의 정보력."

정보력의 의도는 **"같은 돈으로 더 효율적인 선택"**(비싼 게 아니라 잘 아는 것)이다.
**현 구현**은 이 철학을 `studyEfficiencyBonus +0.1`(학업 활동 효율 +10%)로 단순화해 반영한다.
초기 설계의 "히든 선택지(같은 비용 1.3배)·진학 특수 루트·꿀팁 이벤트"는 미구현(§8).

## 8. 미구현 / 보류 (초기 설계 중 현재 코드에 없는 것)

**강점별 초기 스탯 보정 — 대부분 미구현** (실제는 §2 표가 전부)
- 재력 "인기+5", 정보력 "학업+5", 시간자유 "재능+5", 회복력 "인기+5", 엄격함 "체력+3" — 모두 미구현.
- 회복력 초기 스탯은 "학업+8/인기+5/체력+5"가 아니라 **학업+5/체력+5**. 엄격함은 "멘탈-5"가 아니라 **멘탈-3**.

**강점별 고유효과/페널티 — 미구현**
- 회복력 "높은 천장(70+ 감쇠 ×0.65)·외모 이벤트 +30%".
- 재력 "성적 압박·부모 기대치 시스템"(분기 학업 체크·주말 반강제 슬롯) — 이 강제 슬롯/분기 체크 형태는 미구현. (단 strict/info의 진로 기대 갈등은 §5b 이벤트로 구현됨.)
- 정보력 "히든 선택지(1.3배)·간섭 감시·이중생활 발각" — 미구현(효율은 studyEfficiencyBonus +0.1로 대체). 단 info 진로 갈등 아크는 §5b로 구현됨.
- 시간자유 "자립도 ×1.5·알바 조기 해제·자유 루트"(자립도 스탯 자체가 미구현).
- 엄격함 "루틴 보너스 3단계(+1.0/+1.5/+2.0)·루틴 숙련 피로-1·멘탈 소모 1.5배"는 미구현(현재 routineWeeksBoost +1주 가속 + B2 연1회 멘탈+2만). **반항 이벤트는 구현됨**(§5b 사춘기 충돌 — 단 "멘탈30 미만 30% 확률 폭발" 형태가 아니라 Y3 고정주차 아크).
- 정서적 지지의 "번아웃 **2주 단축**·멘탈 긴급 개입(게임당 3회)·부모 대화 항상 +5" 효과. (번아웃 시스템 자체(`burnoutCount`/`burnoutCooldown`)는 02_스탯시스템에 구현됨 — 미구현인 건 emotional 강점의 단축 효과뿐.)

**시스템 미구현**
- 학기 **자유 슬롯 보정/강제 슬롯**(재력 -1, 정보력 +1, 엄격함 -1) — freedom 방학 슬롯 +1만 실재.
- **용돈 차등**(정보력·엄격함 4만원) — "재력 5 / 그 외 3" 단순화.
- **저축 한도**(용돈×8, 재력 40만 등) — money cap 미구현.
- **가정 변동 이벤트**(중반 가정환경 변화).

**구식 표기**
- 피로 자연회복 "-8 고정", 정서적 지지 "고정 +2"(현재 멘탈 70+ 시 +1) 등 — 현 수치는 02_스탯시스템 참조.
- "유전자" 명칭(→ resilience).

## 부록. 조합별 플레이 스타일 (설계 의도)

15조합의 상세 표는 다수가 미구현 효과(성적압박·간섭·자유슬롯)에 기반하므로 제거했다. 현 구현 기준 대표 축만 남긴다:
- **재력 포함**: 용돈 여유(5만) → 유료 활동 폭이 넓다. 단 학원+과외 동시는 여전히 빠듯.
- **정서적 지지 포함**: 멘탈+10 시작 + 피로 회복 +2 → "느리지만 안 무너지는" 안정형.
- **회복력 포함**: 학업·체력 선출발 + 피로 -15% → 갈아넣기 내성.
- **엄격함 포함**: 학업 선출발 + 루틴 가속, 단 멘탈 -3 시작.
- **시간/자유 포함**: 방학 슬롯 +1 + 방치 페널티 절반 → 자율형.
- **정보력 포함**: 학업 활동 효율 +10%.

> 강점=클래스 철학은 유지된다: 모든 조합에 강점과 약점이 공존하고, 자원·멘탈·시간 중 무언가는 반드시 포기해야 한다("포기의 고통").
